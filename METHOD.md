# METHOD.md

## Purpose

This repo uses METHOD to keep Continuum work concrete, inspectable, and bounded.

Continuum work should produce:

- shared coordination doctrine
- explicit ownership maps for cross-repo nouns
- witness and compatibility plans
- integration scenarios
- small design packets with clear playback questions

## Shape

- signposts: [docs/BEARING.md](docs/BEARING.md),
  [docs/VISION.md](docs/VISION.md)
- design cycles: `docs/design/<NNNN-slug>/`
- backlog lanes:
  `docs/method/backlog/inbox/`
  `docs/method/backlog/asap/`
  `docs/method/backlog/up-next/`
  `docs/method/backlog/cool-ideas/`
  `docs/method/backlog/bad-code/`
- retros: `docs/method/retro/`

## Working Rules

1. Start with the smallest honest hill.
2. Prefer one good design packet over scattered notes.
3. Prefer one concrete backlog note over leaving a useful discovery only in
   chat.
4. Keep repo truth short; link outward to active repos instead of copying whole
   worlds into Continuum.
5. Continuum owns coordination truth. Echo, `git-warp`, `warp-ttd`, and Wesley
   still own their local runtime truth.
