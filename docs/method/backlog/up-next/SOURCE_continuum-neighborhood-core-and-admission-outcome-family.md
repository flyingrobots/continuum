# Neighborhood Core And Admission Outcome Family

## Why

Continuum now freezes the shared lawful outcome family in doctrine, and Echo
can already publish truthful runtime mappings for:

- tick admission
- local neighborhood plurality
- settlement publication

But the shared generated-family boundary still lacks one concrete contract slice
that carries those meanings across repos without adapter inference.

## Goal

Freeze one small authored Continuum family for neighborhood-core publication
that includes:

- shared `AdmissionOutcomeKind`
- shared local site plurality
- shared neighborhood participant roles
- one bounded `NeighborhoodCore` publication shape

## Done When

- Continuum has one packet that defines the family boundary
- Continuum has one authored schema for that slice
- the packet explains what remains local versus what becomes shared
- Wesley has a clear compiler-facing target for the next generated family cut
