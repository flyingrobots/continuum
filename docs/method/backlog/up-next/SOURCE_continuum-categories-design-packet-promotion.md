# Continuum Categories Design Packet Promotion

## Why

`docs/continuum-categories.tex` now gives Continuum a source-backed categorical
note for:

- authored contract categories
- host-indexed runtime presentation categories
- realization profunctors
- witness-square conformance
- Wesley's publication-boundary role

That is useful repo truth, but it is still shaped as a standalone TeX/PDF note
rather than a METHOD design packet with playback questions, provenance, and
clear promotion boundaries.

## Goal

Turn the Continuum Categories note into normal Continuum design truth without
inflating it into implementation work.

The promoted packet should preserve the core mathematical claim:

- Continuum owns authored contract meaning
- Wesley compiles host-indexed presentations
- witnesses upgrade generation into conformance claims
- sibling runtimes consume realized surfaces without owning shared contract
  truth

## Non-goals

- implementing a new compiler path
- introducing new schemas only to satisfy the packet
- making Continuum a runtime facade
- deleting the TeX/PDF note before the packet exists

## Done When

- one design packet exists for the categorical story
- it links to `docs/continuum-categories.tex` and
  `docs/continuum-categories.pdf`
- it states what parts are normative doctrine and what parts are mathematical
  scaffolding
- it names follow-up proof obligations without turning them into code in this
  repo
