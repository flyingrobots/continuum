# Doc Governance And Truth Hierarchy

## Why

Old Continuum had a good source-of-truth hierarchy idea, but it was buried in a
dead runtime repo.

## Goal

Rewrite that idea for the active stack:

- METHOD signposts and design packets
- repo-local invariants
- authored contract families
- witness and compatibility truth
- directional notes versus normative notes

## Done When

- Continuum has one short governance note
- it explains precedence and promotion paths
- it helps prevent shadow contracts and duplicate "big picture" docs
